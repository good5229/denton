# 고양·포항 필터 산출물의 원천 범위 기록

## 목적

고양시와 포항시만 따로 잘라 만든 분석 파일은 나중에 다른 지역으로 확장할 때 “원본이 전국 자료였는지, 아니면 고양·포항만 다시 받아야 하는 자료였는지”가 바로 보여야 한다. 이에 따라 현재까지 확인된 주요 필터 산출물에 대해 원천 범위와 재사용 방식을 별도 기록했다.

상세 manifest는 `data/processed/goyang_pohang_filtered_source_coverage_manifest.csv`에 저장했다.

## 핵심 판정

| 구분 | 원천 범위 | 현재 로컬 보관 상태 | 타 지역 확장 방식 |
|---|---:|---|---|
| 건축물대장 표제부 | 전국 | 전국 ZIP 및 전국 변환/필터 산출물 보관 | 같은 원본에서 지역 코드만 바꿔 재추출 |
| 주택 공시가격 | 전국 | 전국 parquet와 고양·포항 parquet 보관 | 전국 parquet에서 재추출 |
| 철도 역별 여객 | 전국 | 전국 파일 보관 후 고양·포항 관련 역 추출 | 역-행정구역 매핑을 바꿔 재추출 |
| 고용·산재보험 사업장 | 전국 | 전국 ZIP 보관 후 고양·포항 주소 필터 | 같은 ZIP에서 주소·지역 조건만 바꿔 재추출 |
| 경기버스 정류소 승하차 | 경기도 전역 | 경기도 ZIP 보관 후 고양시 추출 | 경기도 내 다른 시군은 재추출 가능, 비경기도는 별도 자료 필요 |
| LOCALDATA 인허가 | 전국 서비스 | 현재는 도시별 추출 파일 중심 | 같은 업종을 다른 지역 조건으로 재수집 |
| 건축 인허가 API | 전국 서비스 | 고양·포항 호출분만 보관 | 다른 시군구·법정동 코드로 API 재호출 |
| 부동산 실거래 API | 전국 서비스 | 고양·포항 2024 호출분 중심 | 다른 시군구 코드와 월로 API 재호출 |
| 개인사업자 기본·매출·재무 | 전국 서비스 | 고양·포항 호출분만 보관 | `bizAreaNm` 등 지역 조건으로 API 재호출 |
| 항만 물동량 | 전국 항만 서비스 | 포항항 중심 호출분과 메타데이터 보관 | 다른 항만 코드로 API 재호출 |
| 조달청 입찰공고 | 전국 서비스 | 2023년 1월 일부 페이지만 캐시 | 완결 수집 전까지 분석 원천으로 간주 금지 |
| 금융·보험 집계 | 전국/광역 | `_all.jsonl` 전체 수집 | 시군구 직접 분석에는 제한적, 광역 보조지표로 사용 |

## 특히 주의할 점

1. “전국 원본이 로컬에 있음”과 “API는 전국 조회가 가능하지만 이번 로컬 파일은 고양·포항만 있음”은 다르다.
   - 전국 원본 로컬 보관: 건축물대장, 주택 공시가격, 철도 여객, 고용·산재보험 사업장, 경기버스(경기도 범위).
   - API 재호출 필요: 건축 인허가, 실거래, 개인사업자, 항만 물동량, LOCALDATA 도시별 추출분.

2. 고용·산재보험 사업장 파일은 원본 ZIP을 직접 스캔해 전국 원본임을 확인했다.
   - 스캔 행 수: 2,607,699행
   - 고양·포항 필터 행 수: 70,696행
   - 필터 산출물: `data/processed/phase121_direct_activity_sources/phase121_comwel_workplaces_goyang_pohang_filtered.csv`

3. 조달청 입찰공고는 전국 API이지만 현재 캐시는 부분수집본이다.
   - `data/raw/phase121_direct_activity_sources/pps_2023_ytd/servc_202301_001.json`부터 `010.json`까지만 확인된다.
   - 이 상태를 전국 원본 또는 고양·포항 완결 원본으로 기록하면 안 된다.

4. 개인사업자 자료는 원 서비스의 지역 범위와 현재 로컬 파일 범위를 반드시 분리해서 읽어야 한다.
   - 현재 로컬: `profile/sales/finance_goyang.jsonl`, `profile/sales/finance_pohang.jsonl`
   - 원 서비스: 지역 조건을 바꿔 전국 다른 지역도 조회 가능한 구조
   - 따라서 타 지역 분석 때는 로컬 파일 재사용이 아니라 동일 API 재수집이 필요하다.

## 다음 지역 분석 때의 원칙

1. 먼저 manifest에서 `local_raw_scope`를 확인한다.
2. `전국 원본 보관`이면 지역 코드·주소·역 매핑만 바꿔 재추출한다.
3. `고양·포항 호출분만 보관`이면 원 API/서비스를 같은 방식으로 재호출한다.
4. `부분수집` 또는 `완결 원본 아님`으로 표시된 자료는 성능 검증에 투입하지 않는다.
5. 새 지역 산출물을 만들 때는 같은 manifest에 행을 추가해 “원천 범위 / 로컬 보관 범위 / 재수집 필요 여부”를 남긴다.
