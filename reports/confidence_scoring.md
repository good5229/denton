# 추정치 신뢰등급 산출

## 목적

대시보드와 보고서에서 예측값을 단순히 표시하는 데 그치지 않고, 사용자가 해당 값을 어느 정도 신뢰할 수 있는지 판단할 수 있도록 신뢰등급을 산출했다.

## 산출물

- `scripts/build_confidence_scores.py`
- `data/processed/estimate_confidence_scores.csv`
- `data/processed/estimate_confidence_summary.csv`

CSV는 프로젝트 정책에 따라 CP949로 저장했다.

## 등급 기준

시도 × 산업 × 연도 예측은 `rolling_backtest_error_matrix.csv`의 검증 결과를 사용한다.

주요 입력 지표는 다음과 같다.

| 지표 | 의미 |
|---|---|
| `comparison_count` | 실제값과 비교 가능한 연도 수 |
| `mape` | 개별 연도 평균 절대 백분율 오차 |
| `wmape` | 실제값 규모를 반영한 가중 절대 백분율 오차 |
| `aggregate_percent_error` | 기간 전체 총량 기준 오차율 |

등급은 다음 원칙으로 부여했다.

| 등급 | 해석 |
|---|---|
| A | 낮은 오차와 충분한 backtest 관측치 |
| B | 검증 가능하나 일부 조합에서 오차 관리 필요 |
| C | 탐색적 활용 가능, 지역·산업별 추가 검증 필요 |
| D | 오차가 크거나 검증 근거가 부족해 참고용으로 제한 |

시군구·세부산업·읍면동은 실제 분기 GVA 또는 읍면동 GVA actual이 없으므로 정적 방법론 등급을 별도로 부여했다. 이는 값이 틀렸다는 뜻이 아니라, 검증 가능한 official actual의 해상도가 낮다는 뜻이다.

## 해석

시도 연간 예측은 실제 GRVA와 비교 가능한 backtest가 존재하므로 가장 세밀한 등급을 제공할 수 있다. 반면 시군구 분기, 세부산업, 읍면동은 공식 actual이 부족하므로 `allocation/proxy` 성격을 강하게 표시해야 한다.

따라서 대시보드에서는 선택 조건에 맞는 동적 등급을 우선 표시하되, 없을 경우 수준별 기본 등급을 fallback으로 사용한다.
